Начиная с версии OpenSSL 1.1.1h, проверка для запрета сертификатов в цепочке, которые имеют явно закодированные параметры эллиптической кривой, была добавлена ​​как дополнительная строгая проверка. Ошибка в реализации этой проверки привела к тому, что результат предыдущей проверки для подтверждения того, что сертификаты в цепочке являются действительными сертификатами CA, был перезаписан. Это фактически обходит проверку того, что сертификаты, не являющиеся CA, не должны иметь возможности выдавать другие сертификаты. Если была настроена «цель», то есть последующая возможность для проверки того, что сертификат является действительным CA. Все именованные значения «цели», реализованные в libcrypto, выполняют эту проверку. Поэтому, если установлена ​​цель, цепочка сертификатов все равно будет отклонена, даже если использовался строгий флаг. Цель устанавливается по умолчанию в процедурах проверки сертификатов клиента и сервера libssl, но ее можно переопределить или удалить приложением. BDU:2022-06576 Уязвимость в Python до 3.9.16 (python-3916)
Описание уязвимости
Python 3.9.x до 3.9.16 и 3.10.x до 3.10.9 в Linux допускает локальное повышение привилегий в конфигурации, отличной от конфигурации по умолчанию. Библиотека многопроцессорной обработки Python при использовании с методом запуска forkserver в Linux позволяет десериализовать pickles от любого пользователя в локальном сетевом пространстве имен того же компьютера, что во многих системных конфигурациях означает любого пользователя на том же компьютере. Pickles могут выполнять произвольный код. Таким образом, это допускает локальное повышение привилегий пользователя до пользователя, от имени которого запущен любой процесс forkserver. Установка multiprocessing.util.abstract_sockets_supported в значение False является обходным путем. Метод запуска forkserver для многопроцессорной обработки не является методом запуска по умолчанию. BDU:2022-01315 Уязвимость в Python с 3.8.13 до 3.9.11 (python-3813)
Описание уязвимости
Функция BN_mod_sqrt(), которая вычисляет модульный квадратный корень, содержит ошибку, которая может привести к тому, что она будет зацикливаться вечно для непростых модулей. Внутри себя эта функция используется при разборе сертификатов, содержащих открытые ключи эллиптической кривой в сжатой форме или явные параметры эллиптической кривой с базовой точкой, закодированной в сжатой форме. Можно запустить бесконечный цикл, создав сертификат с недопустимыми явными параметрами кривой. Поскольку разбор сертификата происходит до проверки подписи сертификата, любой процесс, который анализирует внешний сертификат, может, таким образом, подвергнуться атаке типа «отказ в обслуживании». Бесконечный цикл также может быть достигнут при разборе созданных закрытых ключей, поскольку они могут содержать явные параметры эллиптической кривой. Таким образом, уязвимые ситуации включают: - TLS-клиенты, использующие сертификаты сервера - TLS-серверы, использующие клиентские сертификаты - Хостинг-провайдеры, принимающие сертификаты или закрытые ключи от клиентов - Центры сертификации, анализирующие запросы на сертификацию от подписчиков - Все остальное, что анализирует параметры эллиптической кривой ASN.1 Также любые другие приложения, использующие BN_mod_sqrt(), где злоумышленник может контролировать значения параметров, уязвимы к этой проблеме DoS. В версии OpenSSL 1.0.2 открытый ключ не анализируется во время первоначального анализа сертификата, что немного затрудняет запуск бесконечного цикла. BDU:2021-01845 Уязвимость в Python с 3.8.9 до 3.9.4 (python-389)
Описание уязвимости
Флаг X509_V_FLAG_X509_STRICT включает дополнительные проверки безопасности сертификатов, присутствующих в цепочке сертификатов. Он не установлен по умолчанию. Начиная с версии OpenSSL 1.1.1h, проверка для запрета сертификатов в цепочке, которые имеют явно закодированные параметры эллиптической кривой, была добавлена ​​как дополнительная строгая проверка. Ошибка в реализации этой проверки привела к тому, что результат предыдущей проверки для подтверждения того, что сертификаты в цепочке являются действительными сертификатами CA, был перезаписан. Это фактически обходит проверку того, что сертификаты, не являющиеся CA, не должны иметь возможности выдавать другие сертификаты. Если была настроена «цель», то есть последующая возможность для проверки того, что сертификат является действительным CA. Все именованные значения «цели», реализованные в libcrypto, выполняют эту проверку. Поэтому, если установлена ​​цель, цепочка сертификатов все равно будет отклонена, даже если использовался строгий флаг. Цель устанавливается по умолчанию в процедурах проверки сертификатов клиента и сервера libssl, но ее можно переопределить или удалить приложением. Чтобы быть затронутым, приложение должно явно установить флаг проверки X509_V_FLAG_X509_STRICT и либо не указывать цель проверки сертификата, либо, в случае клиентских или серверных приложений TLS, переопределить цель по умолчанию. OpenSSL версий 1.1.1h и более новых подвержены этой проблеме. Пользователи этих версий должны обновиться до OpenSSL 1.1.1k. OpenSSL 1.0.2 не подвержен этой проблеме. Исправлено в OpenSSL 1.1.1k (затронуто 1.1.1h-1.1.1j).

BDU:2024-05196 Уязвимость в Python с 3.8.20 до 3.11.10 (python-3820)
Описание уязвимости
Модуль «ipaddress» содержал неверную информацию о том, были ли определенные адреса IPv4 и IPv6 обозначены как «глобально достижимые» или «частные». Это повлияло на свойства is_private и is_global классов ipaddress.IPv4Address, ipaddress.IPv4Network, ipaddress.IPv6Address и ipaddress.IPv6Network, где значения не возвращались в соответствии с последней информацией из реестров адресов специального назначения IANA. CPython 3.12.4 и 3.13.0a6 содержат обновленную информацию из этих реестров и, таким образом, имеют предполагаемое поведение.
